\section{Data Representation and Preprocessing}\label{sec:method:data}

Operational \gls{elint} recordings are scarce and sensitive, so all training and evaluation data are synthetic.
Pulse trains are produced by Saab's OpenModesty scenario simulator, the same tool used by earlier KTH degree projects on radar pulse data \parencite{jonsson2023supcon}.
OpenModesty specifies flight paths, emitter placements, and operating-mode schedules, then emits and intercepts pulses one at a time.
Receiver-side imperfections (dropped pulses, spurious detections, parameter jitter) are produced inside the simulator and are already present in the exported pulse records, so no separate noise injection is applied downstream.
Each exported pulse carries the continuous fields used here together with the two discrete labels defined in \cref{sec:method:formulation}.

\subsection{Windowing}\label{sec:method:data:window}

Self-attention computes an $L \times L$ compatibility matrix at each layer, so the computational and memory cost of a forward pass grows quadratically with the window length $L$, i.e., $\mathcal{O}(L^2)$. Because a pulse train may contain millions of pulses, processing an entire train at once is infeasible with standard self-attention due to prohibitive memory requirements. While recent work on transformers proposes sub-quadratic attention variants to address this, the present thesis restricts itself to evaluating the baseline (plain) attention mechanism without architectural modifications.

The encoder therefore operates on fixed-length windows of $L = 1024$ pulses.
Given a pulse train $\mat{X} = (\vect{x}_1,\ldots,\vect{x}_N)$ and a stride $s$, the $w$th window is
\begin{equation}\label{eq:method:windowing}
  \mat{X}^{(w)}
  =
  \bigl(\vect{x}_{1 + (w-1)s},\,\ldots,\,\vect{x}_{L + (w-1)s}\bigr),
  \qquad
  w \in \Bigl\{1,\ldots,\bigl\lfloor (N - L)/s \bigr\rfloor + 1\Bigr\}.
\end{equation}
Training uses $s = L/2$, so consecutive windows overlap by half their length; this increases the number of training views per scenario and reduces sensitivity to alignment with mode transitions.
Validation and test windows use $s = L$ so that metrics are not inflated by near-duplicate windows.
The trailing remainder shorter than $L$ is dropped.

At inference, the same fixed-length contract approximates a rolling window over a live pulse stream: a receiver buffers the most recent $L$ pulses, applies the trained encoder, advances by $s$, and repeats.
Training-time windows are therefore representative of the operational reasoning unit, not just a memory convenience.

\subsection{Per-feature scaling}\label{sec:method:data:norm}

Let $\mathcal{P}_{\mathrm{tr}}$ denote the multiset of all pulses in the training split, pooled across pulse trains.
All scaling statistics are estimated on $\mathcal{P}_{\mathrm{tr}}$ and applied unchanged to validation and test pulses.
A per-train fit would normalise out absolute parameter ranges and thereby sharpen contrasts that help intra-train deinterleaving, but operating-mode signatures are defined by those same absolute ranges and are comparable across trains by construction (\cref{sec:method:formulation}); pooling statistics keeps the cross-train scale information that the mode branch relies on, at the price of a softer per-train normalisation for the emitter branch.

For each pulse parameter $g \in \{f,\, \mathrm{PW},\, \log_{10} A\}$, the empirical mean and standard deviation on $\mathcal{P}_{\mathrm{tr}}$ are
\begin{equation}\label{eq:method:standardize_stats}
  \mu_g = \frac{1}{|\mathcal{P}_{\mathrm{tr}}|} \sum_{n \in \mathcal{P}_{\mathrm{tr}}} g_n,
  \qquad
  \sigma_g^2 = \frac{1}{|\mathcal{P}_{\mathrm{tr}}|} \sum_{n \in \mathcal{P}_{\mathrm{tr}}} (g_n - \mu_g)^2,
\end{equation}
and the standardised feature is
\begin{equation}\label{eq:method:standardize}
  \tilde{g}_n = \frac{g_n - \mu_g}{\sigma_g}.
\end{equation}
Amplitude enters the pipeline as $\log_{10} A_n$ rather than $A_n$.
Pulse amplitudes span several orders of magnitude across emitters, ranges, and antenna patterns, and a logarithm compresses that dynamic range so that a few high-power pulses do not dominate the standardised channel.

Time of arrival $t_n$ is min--max scaled to $[0,1]$ against the training-corpus extrema,
\begin{equation}\label{eq:method:minmax}
  \tilde{t}_n
  =
  \frac{t_n - t_{\min}}{t_{\max} - t_{\min}},
\end{equation}
with $t_{\min}, t_{\max}$ taken over $\mathcal{P}_{\mathrm{tr}}$.
The azimuth and lateral components of \gls{aoa}, written $\theta^{\mathrm{az}}_n$ and $\theta^{\mathrm{lat}}_n$, are instead rescaled from their known physical intervals to $[0,1]$ by direct linear mapping:
\begin{equation}\label{eq:method:aoa_rescale}
  \tilde{\theta}^{\mathrm{az}}_n = \frac{\theta^{\mathrm{az}}_n}{2\pi},
  \qquad
  \tilde{\theta}^{\mathrm{lat}}_n = \frac{\theta^{\mathrm{lat}}_n}{\pi},
\end{equation}
since $\theta^{\mathrm{az}}_n \in [0, 2\pi]$ and $\theta^{\mathrm{lat}}_n \in [0, \pi]$ by construction.

Within every window $w$, the scaled \gls{toa} is then offset by the \gls{toa} of the first pulse in that window,
\begin{equation}\label{eq:method:toa_offset}
  \hat{t}_n
  =
  \tilde{t}_n - \tilde{t}_{1 + (w-1)s},
  \qquad
  n \in \{1 + (w-1)s,\,\ldots,\, L + (w-1)s\},
\end{equation}
so that the first pulse of every window has $\hat{t} = 0$.
This removes absolute clock offset while preserving intra-window timing structure; the global min--max step fixes the unit of $\hat{t}$, so $\hat{t}$ differences are comparable across windows.
The \gls{aoa} channels are not offset because their absolute values, not their increments, are physically meaningful.

\subsection{Labels}\label{sec:method:data:labels}

Each pulse is assigned three labels: a train-local emitter identifier, a global operating mode label, and a pulse train ID. Only the emitter and mode labels are used during training; the pulse train ID is included solely for evaluating clustering metrics.

All labels are cast to unique numeric values for consistency and ease of processing.
